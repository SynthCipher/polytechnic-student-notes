PRACTICAL 07

Abstract Methods & Classes

Concept

Abstraction means showing only what is needed and hiding the hard parts. An Abstract Class is like a plan or a map that is not yet a real building.

Real-life Example: When you use a TV remote, you only see the buttons (Change Channel, Volume). You don't need to see the complex wires and chips inside the TV to make it work.

Basic Code Example

// Filename: AbstractDemo.java
abstract class Animal {
    abstract void makeSound(); // No body here
    
    void sleep() {
        System.out.println("Animal is sleeping...");
    }
}

class Dog extends Animal {
    void makeSound() {
        System.out.println("The Dog says: Bark Bark!");
    }
}

public class AbstractDemo {
    public static void main(String[] args) {
        Dog myDog = new Dog();
        myDog.makeSound();
        myDog.sleep();
    }
}

Expected Output

Bark Bark!
Sleeping...

Lab Assignment

Design an abstract class Shape with an abstract method calculateArea(). Create subclasses Circle and Square that implement this method to calculate their respective areas.
