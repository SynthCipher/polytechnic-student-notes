PRACTICAL 14

Modeling Real-world Objects

Concept

A Class is a blueprint (like a drawing of a house), and an Object is the real thing (the actual house built from the drawing).

Real-life Example: A "Car" is a class. It says every car must have 4 wheels and a color. A "Blue Hyundai i20" and a "White Alto 800" are real Objects made from that Car class.

Basic Code Example

// Filename: CarDemo.java
class Car {
    String model, color;
    int year;
    
    // Constructor to initialize object
    Car(String m, String c, int y) {
        model = m; color = c; year = y;
    }
    
    void displayInfo() {
        System.out.println("Model: " + model);
        System.out.println("Color: " + color);
        System.out.println("Year: " + year);
        System.out.println("--------------------");
    }
}

public class CarDemo {
    public static void main(String[] args) {
        Car car1 = new Car("Hyundai i20", "Blue", 2022);
        Car car2 = new Car("Alto 800", "Silver", 2021);
        
        car1.displayInfo();
        car2.displayInfo();
    }
}

Expected Output

Hyundai i20 [Blue] - 2022

Lab Assignment

Consider a Class Car under which Hyundai-i20, Alto-800, and Ignis represent individual Objects. Define properties: Model, Year, Color, Top Speed. Implement methods: Create() to initialize, Sold() to mark status, and display() to show details.
