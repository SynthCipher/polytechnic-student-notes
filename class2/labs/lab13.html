Interfaces & Multiple Inheritance

Concept

An Interface is like a set of rules or a "To-Do List". Any class that uses the interface MUST follow all the rules.

Real-life Example: A "Driver's License" is an interface. It has rules: You must know how to steer, how to brake, and how to park. Any person (class) who wants to be a Driver must follow these rules.

Basic Code Example

// Filename: InterfaceDemo.java
interface Printable { 
    void print(); 
}

interface Showable { 
    void show(); 
}

class Demo implements Printable, Showable {
    public void print() { System.out.println("Printing document..."); }
    public void show() { System.out.println("Showing preview..."); }
}

public class InterfaceDemo {
    public static void main(String[] args) {
        Demo obj = new Demo();
        obj.print();
        obj.show();
    }
}

Expected Output

Printing...
Showing...
