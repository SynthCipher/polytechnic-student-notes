PRACTICAL 15

Employee Management System

Concept

A Base Class has things that are common for everyone. Sub Classes have their own special things. This is like a family tree.

Real-life Example: All "Employees" have a Name and Salary. But a "Software Engineer" writes code, while a "Manager" manages people. They both inherit from Employee.

Basic Code Example

// Filename: EmployeeDemo.java
class Employee {
    String name;
    double salary;
    void work() { System.out.println(name + " is working."); }
}

class Manager extends Employee {
    void manage() { System.out.println(name + " is managing the team."); }
}

public class EmployeeDemo {
    public static void main(String[] args) {
        Manager m = new Manager();
        m.name = "Alice";
        m.salary = 75000;
        
        m.work();   // Inherited method
        m.manage(); // Specific method
    }
}

Expected Output

Employee is working
Manager is managing

Lab Assignment

In a software company, create an Employee base class. Extend it to create classes for: Software Engineer, Sr. Software Engineer, Project Lead, and Director. Each class should have specific properties for their roles and override a getPerks() method to return their specific benefits.
