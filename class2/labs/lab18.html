PRACTICAL 18

Try & Multiple Catch Blocks

Concept

An Exception is a mistake or error that happens when the program is running. We use try-catch to catch these mistakes so the program does not crash.

Real-life Example: Imagine you are driving a car (try). If a tyre punctures (exception), you use a spare tyre (catch) and continue driving. You don't just leave the car in the middle of the road!

Basic Code Example

// Filename: ExceptionDemo.java
public class ExceptionDemo {
    public static void main(String[] args) {
        try {
            int a[] = new int[5];
            // Scenario 1: Division by zero
            // int result = 30 / 0; 
            
            // Scenario 2: Array out of bounds
            a[10] = 50; 
        } 
        catch (ArithmeticException e) {
            System.out.println("Error: Cannot divide by zero!");
        } 
        catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("Error: Array index is out of range!");
        }
        catch (Exception e) {
            System.out.println("Some other error occurred.");
        }
        System.out.println("Program continues...");
    }
}

Expected Output

Arithmetic Error!

Lab Assignment

Write a program to handle exceptions using a try block and multiple catch blocks. Test for ArithmeticException, ArrayIndexOutOfBoundsException, and a general Exception block to catch any other errors.
