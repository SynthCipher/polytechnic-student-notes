PRACTICAL 17

Partial Interface Implementation

Concept

Sometimes a class follows some rules but cannot follow ALL rules of an interface. Such a class becomes Abstract.

Real-life Example: An interface says a "Vehicle" must have a "Brake" and a "Horn". A bicycle has a brake but no horn (it has a bell). So, a bicycle is a partial implementation of the vehicle rules.

Basic Code Example

// Filename: PartialInterfaceDemo.java
interface MyInterface {
    void method1();
    void method2();
}

// This class only implements method1, so it must be abstract
abstract class PartialImpl implements MyInterface {
    public void method1() { 
        System.out.println("Method 1 is implemented here."); 
    }
}

class FullImpl extends PartialImpl {
    public void method2() { 
        System.out.println("Method 2 is implemented here."); 
    }
}

public class PartialInterfaceDemo {
    public static void main(String[] args) {
        FullImpl obj = new FullImpl();
        obj.method1();
        obj.method2();
    }
}

Expected Output

Method 1 done
Method 2 done

Lab Assignment

Write a program that shows the partial implementation of an interface BankOperations which has methods deposit(), withdraw(), and calculateInterest(). Implement only deposit() and withdraw() in an abstract class and the rest in a concrete class.
