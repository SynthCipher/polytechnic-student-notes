Registration, Login & Sessions

Concept

Registration saves a new user. Login checks their password. Sessions help the server "remember" who you are so you don't have to log in on every page.

Real-life Example: Like a hotel key card. You check-in at the desk (Login), they give you a key card (Session), and you can open your room door (Pages) any time.

Auth & Session Code

Filename: auth.php
<?php
session_start();

// 1. Registration
if(isset($_POST['register'])) {
  $user = $_POST['user'];
  $pass = password_hash($_POST['pass'], PASSWORD_DEFAULT);
  mysqli_query($conn, "INSERT INTO users VALUES('$user', '$pass')");
}

// 2. Login
if(isset($_POST['login'])) {
  $user = $_POST['user'];
  $row = mysqli_fetch_assoc(mysqli_query($conn, "SELECT * FROM users WHERE name='$user'"));
  if(password_verify($_POST['pass'], $row['pass'])) {
    $_SESSION['active_user'] = $user;
    header("Location: home.php");
  }
}
?>
